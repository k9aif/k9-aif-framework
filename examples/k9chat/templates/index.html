What does the "Streaming" badge mean?

ON — the answer appears word-by-word as the model generates it, like watching someone type in real time. The model isn't running any faster; you just stop waiting for the whole answer before seeing the first words.

OFF — K9Chat waits for the complete answer, then shows it all at once. Total time to finish is usually about the same either way.

Controlled by chat.stream in config.yaml. Technically: streamed requests go to /chat/stream over Server-Sent Events and call OllamaLLM.generate_stream(); non-streamed requests go to /chat and call generate() once. K9-AIF Execution Pipeline

K9Chat is a reference SBB (Solution Building Block) built on the K9-AIF framework's ABBs (Architecture Building Blocks). This request's path through the stack:

ChatAgent

Extends BaseAgent. Formats prompt + history, calls the model router, persists the new turn.

CacheFactory → BaseCache

Backs conversation memory. In-memory by default; swap to Redis via cache.provider: redis with zero code changes.

K9ModelRouter

Weighted scoring selects the model alias, then dispatches to the LLM adapter — sync or streaming.

OllamaLLM

Concrete BaseLLM adapter. Supports generate() and token-streaming generate_stream().

Session Memory

Each browser tab keeps a session_id. History round-trips through the cache on every turn so follow-up questions keep context.

Governance

Every agent receives a governance pipeline via require_governance() — Noop in dev, enforced in production.
