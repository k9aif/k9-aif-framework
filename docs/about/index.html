About K9-AIF

K9 Agentic Integration Framework — the architectural path, intent, and thinking behind K9-AIF.

How It Started

It began with ideas — not code. In early 2025, Ravi Natarajan started capturing his thinking about how agentic AI systems should be built. Not in a notebook, not in a doc. In a voice recorder on his iPad. Architecture as spoken word — patterns, boundaries, contracts, governance — recorded before a single line was written.

"I wanted to capture my vision of how things have to be done. That capturing reflected into the code — the framework." — Ravi Natarajan

Those voice memos became code. The original intent was focused and specific: build a Retriever — a precise, reliable component that fetches the right information at enterprise scale. That is where the name came from. K9. A retriever. Disciplined, purposeful, trained to find exactly what is needed.

But retrieval exposed deeper questions. What governs what gets fetched? Which agent should act, and when? When should a system decide autonomously, and when should it defer? Those questions were not retrieval questions — they were architecture questions. The codebase grew to answer them.

At some point, Ravi asked ChatGPT directly: "Should this be a framework?" The answer came back: "Absolutely." That was the turning point.

It was not a leap of faith. It was a recognition. Ravi had built frameworks before — in MuleSoft, where the pattern was exactly this: design a template process flow, and let teams plug their business logic in. The framework provides the structure, the governance, the integration plumbing. The team brings the domain. That same principle — applied now to agentic AI, through the lens of TOGAF, OOA/OOD, and UML — became K9-AIF.

For months, every line was written without any AI assistance. Then the tooling evolved: Cursor IDE, using inline code generation (Ctrl+K) for small fragments — a class stub, a method signature. Every fragment reviewed, corrected, made architecturally coherent. The design always came first. When Cursor ran into glitches, he moved to VS Code — and eventually to VS Code with the Claude Code extension, which is where the framework is developed today.

Then, in February 2026, ChatGPT entered the picture — used precisely like hiring a developer. Ravi would write a complete design specification: the pattern, its purpose, how it should be applied, what constraints it must respect. ChatGPT would implement to that specification. The output was reviewed, corrected, adjusted. The architecture was always Ravi's. AI executed; it did not design.

Then came the second shift. All of that hard-won architectural knowledge — the patterns, the rules, the boundaries, the contracts — was captured into two documents: CLAUDE.md and SKILLS.md. Not documentation for humans. Structured context, written so that an AI coding assistant could read it, understand the framework, and work within its architectural boundaries.

"I asked Claude Code to read this and understand the framework. It then started to apply the knowledge to efficiently enhance the framework." — Ravi Natarajan

That is what makes K9-AIF different. The architecture is not just in the code — it is encoded in a form that an AI can reason about and apply correctly. Every building block Claude Code contributes is governed by what Ravi wrote down. The framework grows; the architectural intent never drifts.

More to Come
The ecosystem keeps growing

The rigor K9-AIF carries did not come from a methodology or a textbook. It came from over three decades of building enterprise systems — integration, service-oriented architecture (SOA), object-oriented analysis and design (OOA/OOD), UML, and design patterns. These disciplines were never just about building systems. They were about structuring complexity, defining boundaries, and governing behavior. Applied through a TOGAF-aligned lens to agentic AI, they produced something the industry has been missing: a framework that treats agents as first-class architectural concerns, not just scripts with an LLM attached.

K9-AIF was not created to simply run agents — it was created to structure, govern, and integrate them.

The name stayed — because it still means something.

— Ravi Natarajan
